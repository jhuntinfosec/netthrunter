## Shannon Entropy Deep Dive

### The Formula

$$H = -\sum_{i=1}^{n} P(x_i) \log_2 P(x_i)$$

Where:
- $H$ = entropy in bits
- $P(x_i)$ = probability of character $x_i$ appearing in the string
- $n$ = number of unique characters

### Entropy Ranges for Domain Classification

| Category | Entropy Range | Example | Characteristics |
|----------|--------------|---------|-----------------|
| Short English words | 1.5 - 2.5 | `google`, `apple` | Few unique chars, high repetition |
| Normal domains | 2.5 - 3.2 | `microsoft`, `cloudflare` | Readable words, predictable structure |
| Dictionary-based DGA | 3.0 - 3.8 | `horsestaple`, `reddoorblue` | Real words combined, moderate entropy |
| Random-char DGA | 3.8 - 4.5 | `x18jdfx1zz`, `ab0cx939j` | Random alphanumeric, high entropy |
| Hex/Base64 encoded | 4.0 - 5.0+ | `a3f2c1d8e9b0`, `dGhpcyBp` | Maximum character diversity |

!!! tip "Threshold Selection"
    A threshold of **3.5** catches most random-char DGAs but may miss dictionary-based DGAs. Use **3.2** for higher recall (more false positives) or **3.8** for higher precision (fewer false positives). The capstone project's `--threshold` flag lets you experiment.

### Per-TLD Analysis

Entropy should be calculated on the **root domain only** (strip the TLD). Why:
- `.com` adds fixed characters that reduce entropy artificially
- `.xyz` and `.su` are already suspicious TLDs — entropy of the domain part matters more
- Subdomains should be analyzed separately from the primary domain

### Mathematical Derivation

For a string of length $L$ with $k$ unique characters, maximum entropy is $\log_2(k)$:
- All lowercase (26 chars): max $H \approx 4.7$
- Alphanumeric (36 chars): max $H \approx 5.2$
- Full ASCII printable (95 chars): max $H \approx 6.6$

DGA domains approach maximum entropy for their character set because they're generated by pseudo-random algorithms. Legitimate domains cluster well below maximum because they use recognizable word patterns.

## Feature Engineering for Infrastructure Classification

Entropy alone is a single feature. For robust classification, extract a **feature vector** from each domain:

| Feature | Calculation | What It Reveals |
|---------|-------------|-----------------|
| **Entropy** | Shannon entropy of root domain | Randomness level |
| **Length** | Character count of full domain | DGAs often have fixed lengths |
| **Digit ratio** | digits / total characters | DGAs use more numbers |
| **Consonant ratio** | consonants / alpha characters | Random strings have ~80% consonants |
| **Unique char ratio** | unique characters / length | High for random, low for repetitive |
| **Vowel-consonant ratio** | vowels / consonants | English has ~40% vowels; DGA has ~20% |
| **Max consecutive consonants** | Longest consonant run | English rarely exceeds 3; DGA exceeds 4+ |
| **Has hyphen** | Boolean | Legitimate subdomains use hyphens; pure DGAs rarely do |
| **Numeric suffix** | Ends with digits | Common in DGA variants (`domain123`) |
| **TLD category** | .com/.net/.org = standard; .tk/.su/.xyz = suspicious | Suspicious TLDs correlate with malicious |

```python
# Feature extraction example (from capstone project)
def extract_features(domain: str) -> dict:
    root = domain.split(".")[0]
    alpha = [c for c in root if c.isalpha()]
    vowels = sum(1 for c in root.lower() if c in "aeiou")
    consonants = sum(1 for c in root.lower() if c.isalpha() and c not in "aeiou")

    return {
        "entropy": shannon_entropy(root),
        "length": len(root),
        "digit_ratio": sum(c.isdigit() for c in root) / max(len(root), 1),
        "consonant_ratio": consonants / max(len(alpha), 1),
        "unique_ratio": len(set(root)) / max(len(root), 1),
        "vowel_consonant_ratio": vowels / max(consonants, 1),
    }
```

## K-Means Clustering

### Concept

K-Means groups domains into $K$ clusters based on their feature vectors. Domains with similar characteristics (entropy, length, digit ratio) end up in the same cluster. The result: one cluster for legitimate domains, one for random-char DGAs, one for dictionary DGAs, etc.

### Algorithm Steps

1. **Choose K** — Number of clusters (use elbow method to find optimal K)
2. **Initialize centroids** — Random starting points in feature space
3. **Assign points** — Each domain assigned to nearest centroid (Euclidean distance)
4. **Update centroids** — Move centroid to mean of assigned points
5. **Repeat** steps 3-4 until convergence

### The Elbow Method

Plot **inertia** (sum of squared distances to nearest centroid) vs. K:

```
Inertia
  │
  ├── •
  │     •
  │       •
  │         • ← Elbow (optimal K)
  │           •  •  •  •
  └──────────────────────── K
```

The "elbow" — where inertia stops decreasing dramatically — indicates the optimal number of clusters.

### Silhouette Score

Validates cluster quality. For each point:

$$s(i) = \frac{b(i) - a(i)}{\max(a(i), b(i))}$$

- $a(i)$ = mean distance to points in same cluster
- $b(i)$ = mean distance to nearest other cluster
- $s(i)$ ranges from -1 (wrong cluster) to +1 (well-clustered)

Average silhouette score > 0.5 indicates good clustering. Below 0.3 suggests overlapping clusters.

### Scikit-learn Implementation

```python
from sklearn.cluster import KMeans
from sklearn.preprocessing import StandardScaler
from sklearn.metrics import silhouette_score

# Standardize features (critical — KMeans is distance-based)
scaler = StandardScaler()
X_scaled = scaler.fit_transform(feature_matrix)

# Find optimal K with elbow method
inertias = []
for k in range(2, 10):
    km = KMeans(n_clusters=k, random_state=42, n_init=10)
    km.fit(X_scaled)
    inertias.append(km.inertia_)

# Fit final model
km = KMeans(n_clusters=3, random_state=42, n_init=10)
labels = km.fit_predict(X_scaled)
score = silhouette_score(X_scaled, labels)
```

## Anomaly Detection

### Isolation Forest

Isolation Forest identifies outliers by randomly partitioning feature space. Anomalous points require **fewer partitions** to isolate because they're far from the majority.

**How it works:**
1. Randomly select a feature
2. Randomly select a split value between min and max
3. Repeat until each point is isolated
4. **Anomaly score** = average path length (shorter = more anomalous)

**Why it's good for threat hunting:**
- No need for labeled data (unsupervised)
- Handles high-dimensional feature spaces
- Scales well to large DNS log datasets
- `contamination` parameter controls sensitivity

```python
from sklearn.ensemble import IsolationForest

clf = IsolationForest(
    contamination=0.1,     # Expect ~10% anomalies
    random_state=42,
    n_estimators=100,
)
anomaly_labels = clf.fit_predict(X_scaled)
# -1 = anomaly, 1 = normal
```

### Local Outlier Factor (LOF)

LOF compares the local density of a point to its neighbors. Points in sparse regions (far from neighbors) score as outliers.

- Better than Isolation Forest when clusters have varying densities
- `n_neighbors` controls locality (20 is a good default)
- Computationally heavier than Isolation Forest for large datasets

### Statistical Approaches

For simpler deployments:

| Method | When to Use | Implementation |
|--------|-------------|----------------|
| **Z-score** | Single feature (e.g., entropy only) | Flag domains where $z > 2.5$ |
| **IQR** | Robust to outliers | Flag outside $Q1 - 1.5 \times IQR$ to $Q3 + 1.5 \times IQR$ |
| **Modified Z-score** | Small datasets | Uses median absolute deviation |

## Model Persistence

### Save and Load with Joblib

```python
import joblib

# Save trained model and scaler
joblib.dump(kmeans_model, "domain_classifier.joblib")
joblib.dump(scaler, "feature_scaler.joblib")

# Load in production
model = joblib.load("domain_classifier.joblib")
scaler = joblib.load("feature_scaler.joblib")

# Classify new domains
new_features = extract_features("suspicious-domain.xyz")
scaled = scaler.transform([list(new_features.values())])
cluster = model.predict(scaled)
```

### Retraining Triggers

Retrain the model when:
- False positive rate exceeds threshold (weekly review)
- New DGA family identified (changes feature distribution)
- Domain landscape shifts (new TLDs, IDN adoption changes)
- Model age > 30 days (DNS patterns drift)

## Case Study: Building a DGA Classifier

**Scenario:** Your SOC is seeing thousands of DNS queries per hour to domains that look random. You need an automated classifier to triage them.

### Step 1: Collect Training Data

- **Legitimate domains**: Top 10K from Majestic Million → label = 0
- **DGA domains**: 10K from DGArchive (mixed families: Conficker, Necurs, Qakbot) → label = 1

### Step 2: Feature Extraction

Extract the 6-feature vector (entropy, length, digit ratio, consonant ratio, unique ratio, vowel-consonant ratio) for all 20K domains.

### Step 3: Unsupervised Clustering

Run K-Means with K=3 to see natural groupings:
- **Cluster 0**: Low entropy, short — legitimate
- **Cluster 1**: High entropy, medium length — random-char DGA
- **Cluster 2**: Medium entropy, long — dictionary-based DGA

### Step 4: Anomaly Detection Overlay

Run Isolation Forest (contamination=0.05) to catch domains that don't fit any cluster well — these may be novel DGA families.

### Step 5: Deploy

Save model with joblib. In production, each new DNS query:
1. Extract features
2. Scale with saved scaler
3. Predict cluster → risk score
4. If anomaly → alert SOC

### Step 6: Monitor and Retrain

Track false positive rate weekly. Retrain monthly or when new DGA family is identified.

## Summary

Data science for threat hunting uses three pillars:

1. **Entropy analysis** — Measure randomness to flag machine-generated domains
2. **Clustering (K-Means)** — Group similar infrastructure automatically at scale
3. **Anomaly detection (Isolation Forest)** — Find what doesn't fit known patterns

Combined with feature engineering (entropy + length + digit ratio + consonant patterns), these techniques process millions of DNS records and surface actionable intelligence that manual analysis cannot.
